Run It Locally

pip install -e ".[dev,cli,server,sim]"
vla-zoo predict --model dummy --instruction "hello"
vla-zoo demo pybullet --model dummy --out docs/assets/simulation_pick_place.gif
vla-zoo compare pybullet --models dummy,scripted,random,openvla,pi0,smolvla,groot \
  --out results/vla_runtime_comparison.json \
  --html-out results/vla_runtime_comparison.html
vla-zoo compare dashboard \
  --results results/vla_runtime_comparison.json \
  --out results/vla_runtime_dashboard.html
vla-zoo compare dashboard \
  --status-log examples/ros2/vla_status_sample.jsonl \
  --diagnostics-log examples/ros2/diagnostics_sample.jsonl \
  --out results/vla_ros_runtime_dashboard.html

Adapter Status

Adapter Status Purpose
dummy available No-GPU dry-run adapter for CI, docs, ROS2 launch, and demos.
openvla optional Lazy Hugging Face OpenVLA integration behind vla_zoo[openvla].
pi0, smolvla, groot scaffold Remote-first adapter paths for external heavy VLA stacks.

Safety Boundary

vla_zoo publishes typed action messages. It does not directly command hardware by default. Real robot deployments should add watchdogs, stale-action timeouts, action clipping, and deterministic high-rate controllers downstream of the low-rate VLA policy.

Safety Model
vla_zoo is infrastructure for running and comparing external VLA models. It does not redistribute model weights or claim zero-shot robot success.
